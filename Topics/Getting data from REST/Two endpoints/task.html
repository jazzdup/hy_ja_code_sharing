This problem is fully supported by version 2021.9 of the EduTools plugin and newer. To avoid possible errors, please make sure that the plugin is updated. Alternatively, you can solve it using the web interface.

In this task, you'll create a simple service that returns information about students. The task consists of the following steps:

1. Add a class Student with 2 fields: int id, String name. Add necessary methods and constructors.

2. Create 3 students with arbitrary values. There are two restrictions: the field name shouldn't be null and should contain at least 1 character, and ids should be unique.

3. Implement 2 following endpoints:

  • GET /students: returns information about all 3 students as an array of JSON objects.
  • GET /students/{id}: returns a student with a specified id as a JSON object. If there is no user with a specified id, the program returns null.

Use the same set of students in both methods. The status code should be 200 (OK).

Example 1: Response for a GET /students/84 request:

{
   "id": 84,
   "name": "Alice"
}

Example 2: Response for a GET /students request:

[
    {
        "id": 84,
        "name":"Alice"
    },
    {
        "id": 99,
        "name": "Kate"
    },
    {
        "id": 55,
        "name": "Someone"
    }
]
